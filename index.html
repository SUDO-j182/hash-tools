Why are hashes useful to me?

Password Security

  • You should never store a users actual password - only the hashed version.
  • When a user logs in you hash thier input and compare it to the original stored hash.
  • This means even if your database was leaked, the data is not exposed.

File Integrity Checks

  • Generate a hash for a file and save it.
  • Later regenerate the hash and compare it, if its the same the file has not been altered.
  • Useful for detecting tampering in files

Data Dedublication

  • Hashes help to detect dublicate data quickly.
  • if two documents have the same hash then they are identical.

unique Identifiers

  • Hashes act like digital fingerprints
  • Generate a hash from users data, messages or files to give them a unique ID.

Caching and Performance

  • Uses hashes to check when data changes.
  • if the hash of a file is unchanged, the file is unchanged.

Veryfying Downloads

  • Websites often provide hash keys along with file downloads.
  • You can compare downloaded hash keys with originals.
